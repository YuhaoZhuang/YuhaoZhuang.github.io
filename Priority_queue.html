Priority_Queues

A priority queue has two operations:

Push: add item with associate priority among the list; Pop: remove item with the highest priority among the list.

Priorty queue can be implemented by a Binary Heap

Binary Heap

A Binary Heap is like a binary search tree, with two types of operation:

Min-heap: each node's key is less than or eqal to each of its children. Max-heap: each node's key is greater than or equal to each of its children.

> The lead node doesn't have to be ordered

Root node also always have the highest priority

Pushing values into priority queues: same as the push in binary searach tree, except at the lowest level, doesn't have to care which priority first. Then, while the node has a parent and the parent priority higher than current node, switch these nodes to keep the min or max order remained

Poping values out from priority queues: Same concept as the remove in binary search tree, except keep finding the node's left child, and move that final node to the root. Then keep moving the node to the heap's right subtree to make sure heap order is respected again.
